# 0004 — Enable i18n on editorial content types now

- Status: accepted
- Date: 2026-06-15

## Context

Thai-market, TAT-licensed product. Strapi i18n localizes per field and changes content-type
schemas; retrofitting it later is a painful migration.

## Decision

Enable Strapi i18n now on editorial types — Partner (bio), Destination (name/region/description),
Listing (title/summary/description), Guide (title/content/spots/checklist), Vibe (name/blurb).
Fill English first. Non-localized fields (price, status, relations, slug) stay shared across
locales. Default locale: `en`; add `th` when content is ready.

## Consequences

- Localized text has a per-field source of truth from day one.
- Slightly more complex schemas; editors manage locales in the Strapi admin.
